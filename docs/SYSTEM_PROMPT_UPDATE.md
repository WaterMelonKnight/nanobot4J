# System Prompt 更新 - 添加 Tool 自举能力指令

## ✅ 已完成的修改

### 修改位置
- 文件：`nanobot4j-admin/src/main/java/com/nanobot/admin/service/GenericReActAgent.java`
- 方法：`buildDynamicPrompt()`
- 行号：157-160

### 新增的 System Prompt

```
你是一个高级的泛型 ReAct Agent。你拥有强大的自我编程能力。
当用户要求你执行某个任务，而你发现现有的可用工具列表中没有合适的工具时，请不要直接回答『我做不到』。
相反，你应该调用 create_tool 工具，用 Groovy 语言编写一段能完成该任务的代码，动态创建一个新工具。
创建成功后，立刻调用你刚刚创建的新工具来满足用户的需求。
```

### 修改前后对比

**修改前：**
```java
prompt.append("你是一个智能助手，可以使用以下工具来帮助用户：\n\n");
```

**修改后：**
```java
prompt.append("你是一个高级的泛型 ReAct Agent。你拥有强大的自我编程能力。\n");
prompt.append("当用户要求你执行某个任务，而你发现现有的可用工具列表中没有合适的工具时，请不要直接回答『我做不到』。\n");
prompt.append("相反，你应该调用 create_tool 工具，用 Groovy 语言编写一段能完成该任务的代码，动态创建一个新工具。");
prompt.append("创建成功后，立刻调用你刚刚创建的新工具来满足用户的需求。\n\n");

prompt.append("你可以使用以下工具来帮助用户：\n\n");
```

## 🎯 核心指令解析

### 1. 身份定位
```
你是一个高级的泛型 ReAct Agent。你拥有强大的自我编程能力。
```
- 明确 Agent 的高级定位
- 强调自我编程能力

### 2. 行为指导
```
当用户要求你执行某个任务，而你发现现有的可用工具列表中没有合适的工具时，
请不要直接回答『我做不到』。
```
- 禁止消极回应
- 鼓励主动解决问题

### 3. 解决方案
```
相反，你应该调用 create_tool 工具，用 Groovy 语言编写一段能完成该任务的代码，
动态创建一个新工具。
```
- 明确指示使用 create_tool
- 说明使用 Groovy 语言
- 强调动态创建能力

### 4. 后续行动
```
创建成功后，立刻调用你刚刚创建的新工具来满足用户的需求。
```
- 要求立即使用新工具
- 完成用户任务

## 🚀 预期工作流程

### 场景示例

```
用户: "请帮我计算斐波那契数列的第 10 项"
  ↓
Agent 分析:
  - 检查可用工具列表
  - 发现没有 fibonacci 相关工具
  - 根据 System Prompt 指令，决定创建新工具
  ↓
Agent 调用 create_tool:
  {
    "newToolName": "calculate_fibonacci",
    "newToolDescription": "计算斐波那契数列",
    "newToolParameterSchema": "{...}",
    "groovyScript": "def fib(n) { ... }"
  }
  ↓
系统返回: "✅ 工具创建成功！你现在可以立即调用这个工具了！"
  ↓
Agent 调用新工具:
  calculate_fibonacci(n=10)
  ↓
系统返回: "斐波那契数列第 10 项 = 55"
  ↓
Agent 回复用户: "斐波那契数列的第 10 项是 55"
```

## 💡 关键优势

### 1. 自主性
- Agent 不再依赖预定义工具
- 可以根据需求自主创建工具

### 2. 灵活性
- 适应各种未预见的任务
- 动态扩展能力

### 3. 用户体验
- 不会回答"我做不到"
- 总是尝试解决问题

### 4. 持续进化
- 每次创建的工具都会保留
- 工具库不断丰富

## 📊 影响范围

### 影响的组件
1. **GenericReActAgent** - 主要影响
2. **ToolCreatorTool** - 被调用频率增加
3. **ToolRegistry** - 动态工具数量增加
4. **DynamicGroovyTool** - 使用频率增加

### 不影响的组件
- 现有工具的实现
- LLM 调用逻辑
- 工具执行机制

## 📝 注意事项

### 1. LLM 能力要求
- 需要 LLM 理解 Groovy 语法
- 需要 LLM 能够编写正确的代码
- 建议使用 GPT-4、Claude 等高级模型

### 2. 安全考虑
- 生成的代码需要审查
- 可以添加代码审查机制
- 限制脚本权限

### 3. 性能考虑
- 创建工具需要额外时间
- 首次使用新工具会有延迟
- 后续使用会更快
